# OPM Fingerstyle — Scope and Constraints

## MVP scope
The MVP must include:

### Public pages
- homepage
- guitarist directory
- guitarist profile page
- video listing page
- tablature listing page
- gear page
- article/blog page
- submit profile page
- about page
- contact page
- affiliate disclosure page
- privacy policy page

### Functional areas
- guitarist profile submissions
- admin review and approval
- external YouTube links
- external tablature links
- social links
- gear/product blocks
- affiliate-ready product links
- manual article publishing

## Out of scope for MVP
Do not build:
- promoted profiles
- paid ranking for guitarists
- user comments
- messaging
- creator dashboards
- creator analytics
- automated YouTube ingestion
- direct tab selling
- community posting
- forum/social feed

## Content constraints
- all submissions must be reviewed before publishing
- only relevant OPM/fingerstyle content should be approved
- broken or spammy links must be removable by admin
- empty or thin profiles should not be published

## UX constraints
- profile pages must remain artist-first
- monetization must not dominate profile pages
- sponsored content must be visually distinguishable
- mobile-first layout is required

## Engineering constraints
- solo-founder-friendly
- low-maintenance architecture preferred
- manual moderation first
- simple content model
- use external URLs for videos and tabs in MVP
